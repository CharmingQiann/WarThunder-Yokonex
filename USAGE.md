# 战争雷霆联动使用说明

## 开始使用

1. 点击下方“完成”，返回“插件中心 → 游戏联动”。
2. 找到“战争雷霆”，点击“配置”。
3. 开启联动总开关，并按需启用事件。
4. 为事件选择指令或波形，然后保存。
5. 启动《战争雷霆》，进入试驾或正式战斗。

插件没有单独窗口，保持 GameHub 和联动总开关运行即可。

## 功能

- 空战：过载连续强度、中等/高/极限过载事件、击杀、死亡和坠毁。
- 陆战：速度连续强度、低/中/高速事件、击杀、死亡、维修开始和维修完成。
- 陆战 CAS：进入飞机后自动切换为过载强度，返回坦克后自动恢复速度强度。
- GameHub：离散事件同时进入 IM 指令和本地波形映射；连续强度进入本地设备/手机中继链路。

## 游戏昵称

击杀和死亡识别需要填写游戏昵称：

1. 点击教程下方“在文件夹中打开”。
2. 打开 `config.json`。
3. 把 `player_name` 填成游戏内昵称并保存。
4. 返回 GameHub，重启本地网关。

不填写昵称时，过载、速度、维修和战斗状态联动仍可使用。

## 支持事件

| 事件 | 默认 commandId | 说明 |
| --- | --- | --- |
| 进入战斗时停止 | `_stop_all` | 新战斗开始时清理旧效果 |
| 离开战斗时停止 | `_stop_all` | 返回机库时清理持续效果 |
| 空战中等过载 | `wt_air_g_medium` | 过载进入中等档位 |
| 空战高过载 | `wt_air_g_high` | 过载进入高档位 |
| 空战极限过载 | `wt_air_g_extreme` | 过载进入极限档位 |
| 空战击杀 | `wt_air_kill` | 当前玩家击落或摧毁目标 |
| 空战死亡或坠毁 | `wt_air_death` | 当前玩家被击落或坠毁 |
| 陆战低速行驶 | `wt_tank_speed_low` | 速度进入低速档位 |
| 陆战中速行驶 | `wt_tank_speed_medium` | 速度进入中速档位 |
| 陆战高速行驶 | `wt_tank_speed_high` | 速度进入高速档位 |
| 陆战击杀 | `wt_tank_kill` | 当前玩家摧毁目标 |
| 陆战死亡或被摧毁 | `wt_tank_death` | 当前玩家载具被摧毁 |
| 开始维修 | `wt_tank_repair_start` | 检测到载具开始维修 |
| 维修完成 | `wt_tank_repair_end` | 检测到维修状态结束 |
| 进入陆战空中支援 | `wt_cas_enter` | 陆战中切换到飞机 |
| 离开陆战空中支援 | `wt_cas_exit` | 从飞机返回坦克 |

过载和速度除了档位事件，还会通过 GameHub 本机实时通道连续更新设备输出。

## 强度设置

- 过载、速度、CAS、档位阈值、轮询和防抖规则已经内置。
- 连续强度取对应指令所选波形的 A/B 峰值，再按实时过载或速度比例缩放。
- 强度和 A/B 通道均在 GameHub 波形中设置。
- 不同设备可以为同一指令选择不同波形，各自使用对应的波形强度。

修改事件开关或指令后，通常会在两秒内生效。

## 验证

1. 确认 GameHub 顶部显示“网关运行中”。
2. 确认“战争雷霆”联动总开关已经开启。
3. 启动战争雷霆并进入试驾。
4. GameHub 联动卡片的接收数量应增加。
5. 查看插件目录下的 `logs/war-thunder-yokonex.log`。

## 常见问题

### 没有击杀或死亡事件

检查 `config.json` 中的昵称，必须与 HUD 消息里的名字完全一致。

### 有事件但设备没有动作

检查 GameHub 中事件是否启用、指令 ID 是否存在，以及设备连接页是否已经连接设备。

### 8111 没有数据

先进入一次试驾或正式战斗，再访问 `http://127.0.0.1:8111/state` 检查数据。
